1. How do I approach designing software?

When I approach designing software, I start by planning out the overall structure before writing code. During this project, I learned how important it is to break a scene down into smaller parts, like objects, lighting, and positioning. This helped me stay organized instead of trying to build everything at once.

Some new design skills I developed include understanding how to structure a 3D scene, how to position objects correctly in space, and how to apply transformations like scaling and rotation. I also learned how important it is to think about the visual outcome before coding.

For my design process, I followed a step-by-step approach. I started with basic shapes, then added more details like textures and lighting. Each milestone helped me improve the design little by little.

These design tactics can be used in future work by helping me plan projects ahead of time, stay organized, and build systems in smaller, manageable steps instead of rushing into coding everything at once.

2. How do I approach developing programs?

When developing programs, I focus on building and testing small parts at a time. During this 3D scene project, I used an incremental development strategy, where I added one feature at a time and tested it before moving on.

One important strategy I used was debugging frequently. If something didn’t look right (like objects not appearing or being misplaced), I would go back, check the code, and fix it before continuing.

Iteration played a big role in my development process. I had to constantly adjust object placement, lighting, and scaling to get the scene to look correct. Sometimes things didn’t work the first time, so I had to try multiple times to get the right result.

My approach to coding improved throughout the milestones. At first, I struggled with placing objects and understanding how transformations worked. Over time, I became more confident in adjusting values, organizing code, and fixing issues. By the end of the project, I was able to build a complete 3D scene with better structure and understanding.

3. How can computer science help me in reaching my goals?

Computer science helps me reach my goals by giving me problem-solving skills and technical knowledge that can be applied in many different careers. This project showed me how coding can be used creatively to build visual environments, not just functional programs.

Computational graphics and visualization helped me learn how 3D environments are created and how objects interact in space. This knowledge can help me in future courses where I may work with simulations, game design, or advanced programming concepts.

Professionally, these skills can be useful in areas like software development, game design, or any field that uses visualization tools. Learning how to work with graphics, debug code, and build structured programs gives me a strong foundation for future jobs in tech.
